---

By passing `mediaUrls` (or `imageUrls` for image-only scenarios), different generation modes are automatically selected based on the number and type of materials. Applicable to all `seedance-*` and `jimeng-video-*` models.

<ParamField body="model" type="string" required>
  Model name, e.g. `seedance-2.0-pro`, `seedance-2.0-fast`, `jimeng-video-3.5-pro`.
</ParamField>

<ParamField body="callBackUrl" type="string">
  Webhook callback URL.
</ParamField>

<ParamField body="input" type="object" required>
  <Expandable title="properties" defaultOpen>
    <ParamField body="prompt" type="string" required>
      Prompt. In multi-material mode, use `@1`, `@2`, `@3` to reference the corresponding materials by position.
    </ParamField>
    <ParamField body="mediaUrls" type="string[]">
      List of material URLs. Supports mixed image/video/audio. Media type is auto-detected from the URL extension. The number and type of materials determine the generation mode (see below).
    </ParamField>
    <ParamField body="imageUrls" type="string[]">
      List of reference image URLs (backward compatible, equivalent to `mediaUrls` with images only). If both `mediaUrls` and `imageUrls` are provided, `mediaUrls` takes precedence.
    </ParamField>
    <ParamField body="ratio" type="string">
      Video aspect ratio. Options: `1:1`, `4:3`, `3:4`, `16:9`, `9:16`.
    </ParamField>
    <ParamField body="resolution" type="string">
      Video resolution. Options: `480p`, `720p`, `1080p`.
    </ParamField>
    <ParamField body="duration" type="integer">
      Video duration in seconds. Options: `5`, `10`, `15`.
    </ParamField>
  </Expandable>
</ParamField>

<Note>
  Use `mediaUrls` or `imageUrls` (pick one). Prefer `mediaUrls` for new integrations. If both are provided, `mediaUrls` takes precedence.
</Note>

### Supported Media Types

| Type | Supported Formats | Usage |
|------|-------------------|-------|
| Image | `.jpg`, `.jpeg`, `.png`, `.webp`, `.gif`, `.bmp` | First/last frame or reference image |
| Video | `.mp4`, `.mov`, `.m4v` | Multimodal reference |
| Audio | `.mp3`, `.wav` | Multimodal reference |

<RequestExample>
```bash First Frame (1 image)
curl --request POST \
  --url https://api.maxapi.io/api/v1/task/submit \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
  "model": "seedance-2.0-pro",
  "input": {
    "prompt": "Blow out candles",
    "mediaUrls": ["https://example.com/photo.jpg"]
  }
}'
```

```bash Multimodal (image + audio)
curl --request POST \
  --url https://api.maxapi.io/api/v1/task/submit \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
  "model": "seedance-2.0",
  "input": {
    "prompt": "Use @1 character and @2 background music to generate a video",
    "mediaUrls": [
      "https://example.com/character.jpg",
      "https://example.com/bgm.mp3"
    ]
  }
}'
```
</RequestExample>

<ResponseExample>
```json 200 Submitted successfully
{
  "code": 0,
  "msg": "ok",

    "taskId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
  }
```

</ResponseExample>

## Generation Modes

Different modes are automatically selected based on the number and type of materials:

| Condition | Mode | Description |
|-----------|------|-------------|
| 0 materials | Text-to-Video | Pure text generation |
| 1-2 images only | First/Last Frame | 1st image = first frame, 2nd image = last frame |
| 3+ materials, or contains video/audio | Multimodal Reference | Mixed image/video/audio reference |

### 1 Image — First Frame Mode

The image is used as the first frame of the video, and subsequent animation is generated based on the prompt.

```json
{
  "model": "seedance-2.0-pro",
  "input": {
    "prompt": "Blow out candles",
    "mediaUrls": ["https://assets.movart.ai/landingpage/dog.webp"]
  }
}
```

### 2 Images — First & Last Frame Mode

The first image serves as the video's first frame, the second image as the last frame, and the AI automatically generates the transition animation in between.

```json
{
  "model": "seedance-2.0-pro",
  "input": {
    "prompt": "Transform",
    "mediaUrls": [
      "https://assets.movart.ai/3D-Cartoon.jpg",
      "https://assets.movart.ai/landingpage/dog.webp"
    ]
  }
}
```

### 3+ Images — Multi-Image Reference Mode

Use `@1`, `@2`, `@3` in the prompt to reference the corresponding materials by position.

```json
{
  "model": "seedance-2.0",
  "input": {
    "prompt": "@1 and @2 are chatting, @3 is watching nearby",
    "mediaUrls": [
      "https://assets.movart.ai/3D-Cartoon.jpg",
      "https://assets.movart.ai/landingpage/dog.webp",
      "https://assets.movart.ai/landingpage/dog.webp"
    ]
  }
}
```

### Multimodal Reference (image + video/audio mix)

Seedance 2.0 supports mixed uploads of images, videos, and audio. Media type is auto-detected from the URL extension.

```json
{
  "model": "seedance-2.0",
  "input": {
    "prompt": "Use @1 character and @2 background music to generate a video",
    "mediaUrls": [
      "https://example.com/character.jpg",
      "https://example.com/bgm.mp3"
    ]
  }
}
```

<Note>
  All modes above apply to all `seedance-*` and `jimeng-video-*` models. `imageUrls` can still be used for image-only scenarios (backward compatible). New integrations should use `mediaUrls`.
</Note>

## Query Result

Retrieve the generation result via the [Query Task](/api-reference/query-task) endpoint or Webhook:

```json
{
  "code": 0,
  "msg": "ok",
  "data": {
    "taskId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "status": "SUCCESS",
    "input": {
      "model": "seedance-2.0-pro",
      "prompt": "Blow out candles",
      "mediaUrls": [
        "https://assets.movart.ai/landingpage/dog.webp"
      ]
    },
    "result": {
      "type": "video",
      "urls": [
        "https://v9-dreamnia.jimeng.com/video/tos/cn/video_example.mp4"
      ]
    },
    "created_at": "2026-02-10T23:00:00.000000Z",
    "updated_at": "2026-02-10T23:03:25.000000Z"
  }
}
```
